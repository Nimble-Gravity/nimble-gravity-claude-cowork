Use Cowork Lab

Run a real use case from your industry through start to finish. Pick a task, hand it off on the right model and permission mode, steer the plan, and come away with a deliverable you'd be willing to ship. Along the way, note the part you'd want to turn into a reusable skill.

Exercise · ~40 min
00 — Before you write your first prompt

Five principles from Anthropic's Cowork guidance

The prompts on this page are already written this way — read them and you'll see all five.

  1. Outcome, not steps. Cowork builds its own plan. Describe what the finished output looks like, not the steps to get there.
  2. Say what done looks like. Write your acceptance test before you paste the prompt. If you cannot describe a passing output in one sentence, the prompt is not ready.
  3. Flag gaps, do not fill them. Add a line to every prompt: "If something is missing or unclear, say so — do not estimate." This keeps every figure citable.
  4. Name the reader. "Draft a memo for a credit committee" gets a different output than "summarize this file." The audience shapes the format and detail level. Put it in the prompt.
  5. One sentence beats a rewrite. When the first pass misses, add one corrective sentence and let it rerun. Redirect, do not take over.

01 — Pick your use case

A real job from your own week

Choose something repetitive and heavy on documents that ends in a deliverable. Put the files in a folder, then write the outcome in one sentence along with the test you'll judge it by.

Stage the work

Connect only the folder the task needs. You can seed from an Anthropic industry pack so you start with proven structure. Axos scenario: point Cowork at a commercial loan file (application, financial statements, covenants) and ask for a one-page credit summary with a cited, flagged-risks list. The analyst reviews and signs off — Cowork drafts, it doesn't decide.

Write the acceptance test

Decide what "done" means before you start. You'll reuse this same deliverable and test in Workshop 3 when you turn the job into a repeatable skill.

✅
What a good acceptance test looks like

"The credit memo has a risk section with at least three flagged risks and a recommendation with conditions. Every figure cites a specific line in the loan file. Anything missing or marked interim is called out. Nothing is invented." One or two sentences is enough. If you can read it at the end and know whether Cowork passed or failed, it works. Write it before you start. You'll reuse this same test in Workshop 3 when you turn the job into a skill.

Track 2 — Variance analysis commentary (finance, FP&A, accounting)

Stage one folder with a balance sheet or income statement, a GL detail export or chart of accounts, and a prior period comparison if available. No files? Use the sample variance dataset from the resource library.

Track 2 prompt — paste into Cowork 
Read the financial files in this folder. Draft variance analysis commentary for the balance sheet and income statement line items that show the most material movement. For each variance: name the line item, state the direction and approximate size of the change, and give a plain-language explanation of what is driving it based on what you can see in the files. If a prior period comparison is missing or a figure looks incomplete, say so and do not fill it in. Format the output as a commentary section ready for a finance reviewer to edit and sign off on.

Acceptance test: "The commentary covers at least three material line item variances, explains the direction and size of each movement in plain language, and flags any line where the source data is incomplete or where a prior period comparison is missing. No numbers are invented."

02 — Hand it off and steer

Sonnet, ask before acting, read the plan

Put Lesson 2 into practice. Start on Sonnet, keep approvals on, and read the plan before you let it run. Track what the task took so the value is easy to point to later.

Do

Hand off the whole task

Describe the outcome, let Cowork plan across the folder, approve the steps, and redirect with a sentence if it drifts. Don't reach in and take it over.

Direct the work; let Cowork do it.
Record

Time, rounds, and quality

Track three things while you run: how long the task took from start to finished deliverable, how many rounds of feedback it needed before the output passed your acceptance test, and how close the first pass came — would you have sent it as-is, or did it need a full revision? Write these down before you leave the lab. They are what you bring to Workshop 4 to show the value of this workflow and to seed the adoption dashboard. A finished deliverable with those three numbers is worth more than a dozen chat transcripts.
